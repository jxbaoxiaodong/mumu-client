您当前的 AI 配额已使用完毕，以下为您提供解决方案

方案一：自己配置 API（推荐）

如果您有自己的 AI API Key，可以配置本地客户端直接调用，无需使用系统配额。

优势：
  • 无配额限制，想用多少用多少
  • 数据不经过服务器，隐私更安全
  • 支持多种 AI 服务商（OpenAI、通义千问、DeepSeek 等）
操作步骤：
1. 获取源码

从 GitHub 下载客户端源码

git clone https://github.com/你的仓库地址/mumu.git
2. 修改配置文件

编辑 data/api_config.json，填入您的 API 信息：

{
  "text_model": {
    "api_token": "您的API密钥",
    "api_url": "https://api.openai.com/v1",
    "model_name": "gpt-4o-mini"
  },
  "vision_model": {
    "api_token": "您的API密钥",
    "api_url": "https://api.openai.com/v1",
    "model_name": "gpt-4o-mini"
  },
  "speech_model": {
    "api_token": "您的API密钥",
    "api_url": "https://api.openai.com/v1",
    "model_name": "whisper-1"
  }
}
3. 本地运行

安装依赖并启动客户端：

cd mumu
pip install -r requirements.txt
python client_public_final.py
